🔐 Security Architecture

Is system mein vote manipulation mathematically impossible hai

🔑

WebAuthn / FIDO2

Fingerprint ka image kabhi server par nahi jaata — sirf cryptographic proof. Biometric data phone ke andar sealed rehta hai.

🧱

Privacy Wall

VotersCollection aur BallotCollection mein zero common fields. Admin vote trace nahi kar sakta — mathematically impossible.

One-Time Token

🔄 Data Flow Architecture

1️⃣
Registration: Aapka naam + voterID + zone + fingerprint publicKey server par save hota hai. Raw fingerprint kabhi store nahi hoti.
2️⃣
Authentication: Server ek challenge bhejta hai. Aapka phone challenge ko private key se sign karta hai. Server public key se verify karta hai. Match = token issue.
3️⃣
Anonymous Token: Token mein sirf { sessionId + zone } hota hai. Naam, voterID, aadhaar — kuch nahi. Token 10 minute mein expire.
4️⃣
Ballot: Vote cast hone par sirf { candidateId + zone + timestamp } save hota hai. Voter identity kabhi ballot mein nahi aati.

🤔 Fingerprint Server Par Kyun Nahi Jaati?

WebAuthn (FIDO2 standard) use karta hai public-key cryptography. Is mein ek mathematical principle hai:

🔑 PRIVATE KEY

Phone ke andar secure chip mein sealed. Kabhi nahi nikalti. Fingerprint se unlock hoti hai.

🔓 PUBLIC KEY

Server par save hoti hai. Sirf verify karne ke kaam aati hai. Fingerprint recreate karna impossible.

UIDAI breach → Sirf public keys leak hoongi. Fingerprint images = 0. Vote manipulation = mathematically impossible.
